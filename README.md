# Generative AI Research Project

This repository is used for organising research materials for the REIT6811 project: "Using Generative AI Tools - Boon or Bane".

## Repository Structure

- `draft/`: Stores draft research proposals, draft reports, and working versions of written documents.
- `for analysis/`: Stores files used for quantitative or qualitative analysis, such as data files, analysis scripts, tables, and figures.
- `maybe can use for analysis/`: Stores additional materials that may be useful for later analysis or comparison.
- `sources/`: Stores literature review sources, background readings, articles, and supporting references.
- `README.md`: Provides an overview of this repository and guidelines for using it.

## How to Navigate the Files

Start with the `sources/` folder to review background materials and literature. Then use the `for analysis/` folder to locate data, scripts, and analysis-related files. Draft documents and reports can be found in the `draft/` folder.

## Naming Convention

Files should be named clearly and consistently so that team members can identify the content, purpose, and version of each file. A recommended naming structure is:

`YYYYMMDD_ProjectName_FileType_Author_Version`

Example:

`20260427_GenerativeAI_SurveyAnalysis_Irene_v1.docx`

## Collaboration Guidelines

Future collaborators should use clear file names, write meaningful commit messages, and avoid uploading unnecessary duplicate files. Any major change should be committed with a short message explaining what was changed.

## Data Management Note

GitHub is useful for organising documents, code, and version history, but sensitive participant-related data should not be uploaded to GitHub in a real research project. Files such as raw survey responses, interview transcripts, consent forms, and identifiable photos should be stored securely in UQRDM or UQ Research Drive with restricted access.
